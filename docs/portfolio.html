Architecture

🔐 Auth & Sécurité

JWT httpOnly cookies — aucun token côté client, refresh transparent.

🍪

httpOnly Cookies

Access token (30 min) + Refresh token (7 jours) stockés en cookie httpOnly. Inaccessibles depuis JS — protection XSS.

♻️

Refresh silencieux

Le client intercepte les 401, appelle /auth/refresh et rejoue la requête échouée. File d'attente pour les requêtes parallèles.

🛡️

Secrets distincts

Access JWT secret et Refresh JWT secret indépendants en production — rotation séparée possible. Validation à la startup.

🚦

Rate limiting

Middleware custom sur les endpoints sensibles (login, reset password). Retour 429 avec header Retry-After.

📧

Activation email

Token signé à usage unique pour l'activation de compte et la réinitialisation de mot de passe (SMTP).

📊

Activity logs

Middleware qui trace chaque requête (méthode, endpoint, durée, code HTTP, user). Dashboard admin avec stats.
